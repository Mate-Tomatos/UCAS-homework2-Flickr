## 一、任务目标

本次作业要求在 Flickr30k 数据集上复现《Visual Semantic Reasoning for Image-Text Matching》中的 VSRN 模型，并以图文检索召回率作为评价指标。作业加分项要求自行选择较新的图文检索模型，在 Flickr30k 上测试模型效果。

本文完成两组实验：第一组为 VSRN 在 Flickr30k `f30k_precomp` 区域特征上的复现实验；第二组为 CLIP `openai/clip-vit-base-patch32` 在 Flickr30k test split 上的零样本检索实验。

## 二、数据集与评价指标

实验使用 Flickr30k 数据集。每张图片对应 5 条英文描述。数据划分采用 Karpathy split：训练集 29,000 张图片、验证集 1,014 张图片、测试集 1,000 张图片。对应 caption 数量分别为 145,000、5,070 和 5,000。

VSRN 实验使用 SCAN/Bottom-Up Attention 提供的预计算区域特征，每张图片包含 36 个区域，每个区域为 2048 维特征。特征文件检查结果如下：`train_ims.npy` 为 `(29000, 36, 2048)`，`dev_ims.npy` 为 `(5070, 36, 2048)`，`test_ims.npy` 为 `(5000, 36, 2048)`。

评价指标为 Recall@K，包括图搜文 Image-to-Text Recall@1/5/10 和文搜图 Text-to-Image Recall@1/5/10。Recall@K 表示正确匹配结果出现在排序前 K 个候选中的比例。另报告 RSum，即六个 Recall 指标之和。

## 三、VSRN 复现实验

### 3.1 方法简介

VSRN 的核心思想是对图像区域特征建立视觉语义关系图，通过图卷积或关系推理增强区域级视觉表示，再与文本编码器输出映射到统一语义空间中。训练目标采用图文匹配排序损失，并结合 caption 相关辅助损失。与直接使用整图 CNN 特征的方法相比，VSRN 显式利用区域之间的关系结构，更适合细粒度图文匹配。

### 3.2 复现设置

代码基于官方仓库 `KunpengLi1994/VSRN`。官方代码原始依赖 Python 2.7 和 PyTorch 0.4.1，但该旧环境在当前 A100/CUDA 机器上存在 cuDNN 版本和硬件兼容问题。因此本实验在现代 PyTorch 环境 `/mnt/kxh/miniconda3/envs/trl` 中运行，并对代码做必要兼容修复。

关键训练参数如下：

| 参数 | 取值 |
| --- | --- |
| 数据 | Flickr30k `f30k_precomp` |
| batch size | 128 |
| embedding size | 2048 |
| word dim | 300 |
| learning rate | 0.0002 |
| margin | 0.2 |
| max violation | True |
| epoch | 30 |
| lr_update | 10 |
| max_len | 60 |

训练命令如下：

```bash
cd /mnt/kxh/smx/homework/h2
CUDA_VISIBLE_DEVICES=6 bash scripts/experiments/run_vsrn_train_f30k.sh flickr_VSRN --val_step 100000
```

其中 `--val_step 100000` 用于关闭 epoch 内频繁验证，保留每个 epoch 结束后的验证与 checkpoint 保存，不改变训练损失或优化过程。

### 3.3 兼容性处理

复现过程中主要处理了以下工程问题：

1. 旧版 PyTorch 0.4.1 与当前 A100/CUDA 环境存在 cuDNN 版本冲突，最终改用现代 PyTorch 环境运行。
2. Python 2 文本处理中的 `.decode('utf-8')` 在 Python 3 下失效，改为统一文本转换函数。
3. `.iteritems()`、`/` 除法、`loss.data[0]` 等旧 API 分别替换为 Python 3 和现代 PyTorch 兼容写法。
4. 预计算特征较大，改用 `np.load(..., mmap_mode='r')`，避免一次性加载 8.5GB 训练特征。
5. Caption 分词结果在 Dataset 初始化时缓存，避免训练过程中重复分词。
6. `pycocotools` 对 `f30k_precomp` 分支不是必需依赖，因此改为可选导入。

## 四、加分项：CLIP 零样本检索

### 4.1 方法简介

加分项选择 CLIP `openai/clip-vit-base-patch32`。CLIP 通过大规模图文对比学习训练图像编码器和文本编码器，可直接在未微调的情况下进行零样本图文检索。实验中对 Flickr30k test split 的图片和 caption 分别编码，使用归一化向量内积作为相似度，并计算双向 Recall@K。

### 4.2 运行命令

```bash
cd /mnt/kxh/smx/homework/h2
CUDA_VISIBLE_DEVICES=6 bash scripts/experiments/run_clip_f30k_test.sh
```

### 4.3 实验结果

| 模型 | I2T R@1 | I2T R@5 | I2T R@10 | T2I R@1 | T2I R@5 | T2I R@10 | RSum |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| CLIP ViT-B/32 zero-shot | 79.40 | 95.00 | 98.10 | 58.84 | 83.46 | 90.04 | 504.84 |

CLIP 在未使用 Flickr30k 训练数据的零样本设定下取得较强结果，说明大规模预训练模型已经学习到较通用的图文语义对齐能力。

## 五、分析与讨论

VSRN 和 CLIP 代表两类不同的图文检索思路。VSRN 使用预检测区域特征，并通过图结构建模区域关系，其优势在于显式利用目标级视觉关系；CLIP 则依赖大规模图文对比预训练，不显式构建区域图，但具备强零样本泛化能力。

从实验结果看，CLIP 零样本 RSum 为 504.84，高于 VSRN 的 466.30，说明大规模图文预训练在 Flickr30k 上具有较强迁移能力。复现 VSRN 的主要难点不在算法本身，而在旧版深度学习框架与当前 GPU 环境之间的兼容。为保证实验可运行，本次复现保留官方模型结构和训练协议，并仅进行必要的 Python 3 与现代 PyTorch 兼容修复。CLIP 加分项则依赖现代 Transformers 工具链，实验流程更稳定，且无需训练即可得到较高 Recall。
