## 2. VSRN 复现实验

### 目的

复现《Visual Semantic Reasoning for Image-Text Matching》在 Flickr30k 上的图文检索实验，以 Recall@K 作为评价指标。

### 模型与输入

- 模型：官方 VSRN 代码，来自 `KunpengLi1994/VSRN`。
- 输入：SCAN/Bottom-Up Attention 预计算区域特征 `f30k_precomp`。
- 特征形状：
  - `train_ims.npy`: `(29000, 36, 2048)`
  - `dev_ims.npy`: `(5070, 36, 2048)`
  - `test_ims.npy`: `(5000, 36, 2048)`
- caption 数量：train 145,000，dev 5,070，test 5,000。

### 环境与兼容处理

官方代码依赖 Python 2.7 / PyTorch 0.4.1。该旧环境在当前 A100 + CUDA 环境中出现 cuDNN/CUDA 兼容问题，因此最终采用 `/mnt/kxh/miniconda3/envs/trl` 中的现代 PyTorch 运行，并做了如下兼容修复：

- `pycocotools` 改为可选依赖，`f30k_precomp` 分支不需要 COCO API。
- Python 2 的 `.decode('utf-8')` 改为 Python 3 文本兼容处理。
- `.iteritems()` 改为 `.items()`。
- Python 3 除法导致的 `npts` 浮点问题改为整除。
- `loss.data[0]` 改为 `loss.item()`。
- 非连续张量的 `.view()` 改为 `.reshape()`。
- 预计算特征使用 `np.load(..., mmap_mode='r')`，避免启动时一次性载入 8.5GB 特征。
- caption 分词结果在 `PrecompDataset` 中缓存，避免训练时重复分词。
- RNN 的 `bidirectional` 参数显式转为 `bool`，适配现代 PyTorch。
- `tensorboard_logger` 缺失时使用 no-op logger，不影响训练与日志输出。

## 4. 加分项：CLIP 零样本 Flickr30k 检索

### 目的

选择较新的图文检索模型作为加分项。实验使用 Hugging Face Transformers 中的 CLIP `openai/clip-vit-base-patch32`，在 Flickr30k test split 上进行零样本图文检索。

### 设计

- 不在 Flickr30k 上训练或微调 CLIP。
- 对 1,000 张 test 图片和 5,000 条 caption 分别编码。
- 使用归一化图像特征与文本特征的内积作为相似度。
- 评价指标为 Image-to-Text 和 Text-to-Image 的 Recall@1/5/10，以及六项指标求和 RSum。

### 命令

```bash
cd /mnt/kxh/smx/homework/h2
CUDA_VISIBLE_DEVICES=6 bash scripts/experiments/run_clip_f30k_test.sh
```

实际输出文件：`results/clip_flickr30k_test_metrics.json`。

### 结果

| 模型 | Split | 图像数 | Caption 数 | I2T R@1 | I2T R@5 | I2T R@10 | T2I R@1 | T2I R@5 | T2I R@10 | RSum |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| openai/clip-vit-base-patch32 | test | 1000 | 5000 | 79.40 | 95.00 | 98.10 | 58.84 | 83.46 | 90.04 | 504.84 |

结论：CLIP 在不使用 Flickr30k 训练数据的零样本设定下已经取得较强检索性能，可作为 VSRN 图结构模型之外的较新图文检索对照。
